📅 Semana 3 - Lo que aprendí
Esta semana aprendí a usar operadores en JavaScript para hacer cálculos, comparaciones y validaciones lógicas dentro de mi dominio de tienda de productos orgánicos y Naturales.
Qué aprendí

+, -, *, /, %, ** → los operadores aritméticos sirven para calcular totales, promedios, residuos y potencias
+=, -=, *=, /= → la asignación compuesta permite actualizar un valor acumulado sin repetir la variable
=== → compara valor y tipo de dato al mismo tiempo, es la forma correcta y estricta de comparar
!== → verifica que dos valores sean diferentes, también de forma estricta
== → nunca se debe usar porque puede dar resultados inesperados al comparar tipos distintos
&& → operador lógico "Y", ambas condiciones deben ser verdaderas
|| → operador lógico "O", basta con que una condición sea verdadera
! → operador lógico "NO", invierte el resultado de una condición

Qué hice
🧐 Hice una calculadora de dominio para una tienda de productos orgánicos y Naturales que calcula el total de ventas, aplica descuentos y cupones sobre un carrito acumulado, valida el método de pago y el estado del stock con === y !==, y combina condiciones lógicas para determinar si aplica envío gratis o si un producto está disponible para venta.
Lo que no sabía antes y ahora sí sé
Que == y === no son lo mismo: con == JavaScript intenta convertir los tipos antes de comparar y eso puede generar errores silenciosos, mientras que === compara exactamente sin ninguna conversión. También aprendí que la asignación compuesta no es solo un atajo, sino que hace el código más legible cuando se trabaja con valores que se van actualizando paso a paso.
